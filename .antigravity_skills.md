# Regras de Desenvolvimento - Hub Dufrio

Para garantir a consistência do projeto e o fluxo de trabalho do Thiago Luz, as seguintes regras devem ser seguidas em **todas** as sessões de desenvolvimento:

## 1. Atualização Obrigatória do Index
Sempre que uma alteração for feita no projeto e enviada para o GitHub, o arquivo `index.html` **DEVE** ser atualizado primeiro.
- Localize o objeto `updatesData` no script do `index.html`.
- Adicione uma nova entrada no topo do array `updates`.
- Campos obrigatórios: `date` (DD/MM/AAAA), `time` (HH:MM), `title` e `description`.

## 2. Mensagem de Atualização (Git)
- O arquivo `mensagem_atualizacao.txt` deve ser atualizado com um resumo de uma linha antes de rodar o commit.
- Use sempre o script `Enviar_GitHub.bat` para realizar o push, garantindo que o fluxo de automação do Thiago seja respeitado.

## 3. Preservação de Arquivos Cruciais
- **NÃO EXCLUIR**: `worker-novo.js` (infraestrutura do proxy).
- **NÃO EXCLUIR**: `mensagem_atualizacao.txt` (automação do git).
- **NÃO EXCLUIR**: `Enviar_GitHub.bat` e `Atualizar_do_GitHub.bat`.
